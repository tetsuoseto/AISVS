# C1 학습 데이터 거버넌스 및 편향 관리

## 제어 목표

학습 데이터는 출처, 보안, 품질, 공정성을 보존하는 방식으로 수집, 처리 및 유지되어야 합니다. 이렇게 함으로써 법적 의무를 이행하고, AI 수명 주기 전반에 영향을 미칠 수 있는 학습 중 발생하는 편향, 오염 또는 개인정보 침해 위험을 줄일 수 있습니다.

---

## C1.1 학습 데이터 출처

모든 데이터셋의 검증 가능한 재고를 유지하고, 신뢰할 수 있는 출처만 허용하며, 감사를 위해 모든 변경 사항을 기록하십시오.

|   #   | 설명                                                                                               | 레벨  | 역할  |
| :---: | ------------------------------------------------------------------------------------------------ | :-: | :-: |
| 1.1.1 | 모든 훈련 데이터 소스(출처, 관리자/소유자, 라이선스, 수집 방법, 의도된 사용 제약 조건 및 처리 이력)의 최신 인벤토리가 유지되고 있는지 확인하십시오.          |  1  | D/V |
| 1.1.2 | 학습 데이터 처리 과정에서 불필요한 특징, 속성 또는 필드(예: 사용되지 않는 메타데이터, 민감한 개인식별정보(PII), 누출된 테스트 데이터)가 제외되었는지 확인하십시오. |  1  | D/V |
| 1.1.3 | 모든 데이터셋 변경 사항이 기록된 승인 워크플로우의 대상임을 확인하십시오.                                                        |  2  | D/V |
| 1.1.4 | 가능한 경우 데이터셋 또는 하위 집합에 워터마크 또는 지문이 포함되어 있는지 확인하십시오.                                               |  3  | D/V |

---

## C1.2 훈련 데이터 보안 및 무결성

학습 데이터에 대한 접근을 제한하고, 저장 시 및 전송 중에 이를 암호화하며, 변조, 도난 또는 데이터 오염을 방지하기 위해 무결성을 검증하십시오.

|   #   | 설명                                                                              | 레벨  | 역할  |
| :---: | ------------------------------------------------------------------------------- | :-: | :-: |
| 1.2.1 | 접근 제어가 학습 데이터 저장소 및 파이프라인을 보호하는지 확인하십시오.                                        |  1  | D/V |
| 1.2.2 | 모든 훈련 데이터 접근이 사용자, 시간, 행동을 포함하여 기록되는지 확인하십시오.                                   |  2  | D/V |
| 1.2.3 | 교육 데이터셋이 전송 중 및 저장 시 산업 표준 암호화 알고리즘과 키 관리 방식을 사용하여 암호화되어 있는지 확인하십시오.            |  2  | D/V |
| 1.2.4 | 학습 데이터 저장 및 전송 중에 데이터 무결성을 보장하기 위해 암호화 해시 또는 디지털 서명이 사용되는지 확인하십시오.              |  2  | D/V |
| 1.2.5 | 훈련 데이터의 무단 변경 또는 손상을 방지하기 위해 자동화된 감지 기술이 적용되었는지 확인하십시오.                         |  2  | D/V |
| 1.2.6 | 오래된 학습 데이터가 안전하게 삭제되거나 익명화되었는지 확인하십시오.                                          |  2  | D/V |
| 1.2.7 | 모든 학습 데이터셋 버전이 고유하게 식별되고, 변경 불가능하게 저장되며, 롤백 및 포렌식 분석을 지원할 수 있도록 감사 가능함을 검증하십시오. |  3  | D/V |

---

## C1.3 학습 데이터 라벨링 품질, 무결성 및 보안

라벨을 보호하고 중요한 데이터에 대해 기술 검토를 요구합니다.

|   #   | 설명                                                                                        | 레벨  | 역할  |
| :---: | ----------------------------------------------------------------------------------------- | :-: | :-: |
| 1.3.1 | 라벨 아티팩트의 무결성과 진위를 보장하기 위해 암호화 해시 또는 디지털 서명이 적용되었는지 확인합니다.                                 |  2  | D/V |
| 1.3.2 | 라벨링 인터페이스와 플랫폼이 강력한 접근 제어를 시행하고, 모든 라벨링 활동에 대한 변조 방지 감사 로그를 유지하며, 무단 수정으로부터 보호하는지 확인하십시오. |  2  | D/V |
| 1.3.3 | 라벨의 민감한 정보가 저장 시 및 전송 시 데이터 필드 수준에서 삭제, 익명화 또는 암호화되었는지 확인하십시오.                            |  3  | D/V |

---

## C1.4 학습 데이터 품질 및 보안 보증

데이터 세트 신뢰성을 보장하기 위해 자동화된 검증, 수동 샘플 검사 및 기록된 수정 조치를 결합하세요.

|   #   | 설명                                                                                                                                                                                 | 레벨  | 역할  |
| :---: | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :-: | :-: |
| 1.4.1 | 자동화된 테스트가 모든 인제스트 또는 중요한 데이터 변환 시 형식 오류와 널(null) 값을 잡아내는지 확인하십시오.                                                                                                                  |  1  |  D  |
| 1.4.2 | LLM 교육 및 미세 조정 파이프라인이 잠재적인 중독 공격(예: 레이블 뒤집기, 백도어 트리거 삽입, 역할 전환 명령, 영향력 있는 인스턴스 공격) 또는 훈련 데이터 내 의도치 않은 데이터 손상을 식별하기 위해 중독 탐지 및 데이터 무결성 검증(예: 통계적 방법, 이상치 탐지, 임베딩 분석)을 구현하는지 확인하십시오. |  2  | D/V |
| 1.4.3 | 자동 생성된 라벨(예: LLM 또는 약한 감독을 통해 생성된 라벨)이 환각, 오도되었거나 신뢰도가 낮은 라벨을 감지하기 위해 신뢰도 임계값 및 일관성 검사를 적용받는지 확인하십시오.                                                                              |  2  | D/V |
| 1.4.4 | 적절한 방어 수단이 구현되고 조정되었는지 확인하십시오. 예를 들어, 적대적 훈련(생성된 적대적 예제를 사용), 교란된 입력을 포함한 데이터 증강, 또는 견고한 최적화 기법 등이 있으며, 이러한 방어 수단은 위험 평가에 기반하여 관련 모델에 적용되어야 합니다.                                   |  3  | D/V |
| 1.4.5 | 자동화된 테스트가 모든 수집 또는 중요한 데이터 변환 시 라벨 왜곡을 포착하는지 확인하십시오.                                                                                                                               |  3  |  D  |

## C1.5 데이터 계보 및 추적 가능성

감사 추적 및 사고 대응을 위해 각 데이터 포인트가 소스에서 모델 입력까지의 전체 경로를 추적합니다.

|   #   | 설명                                                                                                              | 레벨  | 역할  |
| :---: | --------------------------------------------------------------------------------------------------------------- | :-: | :-: |
| 1.5.1 | 각 데이터 포인트의 계보와 모든 변환, 증강 및 병합이 기록되어 있으며 재구성할 수 있는지 확인하십시오.                                                      |  2  | D/V |
| 1.5.2 | 계보 기록이 변경 불가능하며, 안전하게 저장되고, 감사 목적으로 접근 가능함을 확인하십시오.                                                             |  2  | D/V |
| 1.5.3 | 프라이버시 보호 또는 생성 기법을 통해 생성된 합성 데이터에 대해 계보 추적이 적용되는지 확인하고, 모든 합성 데이터가 전체 파이프라인에서 명확하게 표시되어 실제 데이터와 구분될 수 있도록 하십시오. |  2  | D/V |
