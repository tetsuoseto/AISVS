# AI 구성 요소 및 사용자를 위한 C5 접근 제어 및 신원 관리

## 제어 목표

AI 시스템에 대한 효과적인 접근 제어는 강력한 신원 관리, 컨텍스트 인식 권한 부여 및 제로 트러스트 원칙에 따른 실행 시간 강제 적용을 필요로 합니다. 이러한 제어는 인간, 서비스 및 자율 에이전트가 명시적으로 부여된 범위 내에서만 모델, 데이터 및 계산 자원과 상호작용하도록 보장하며, 지속적인 검증 및 감사 기능을 포함합니다.

---

## C5.1 신원 관리 및 인증

다단계 인증을 통해 모든 엔티티에 대해 암호학적으로 지원되는 신원을 구축하십시오.

|   #   | 설명                                                                                        | 레벨  | 역할  |
| :---: | ----------------------------------------------------------------------------------------- | :-: | :-: |
| 5.1.1 | 모든 인간 사용자와 서비스 프린서플이 OIDC 및/또는 SAML 프로토콜을 사용하여 중앙 집중식 기업 아이덴티티 제공자(IdP)를 통해 인증하는지 확인하십시오. |  1  | D/V |
| 5.1.2 | 고위험 작업(모델 배포, 가중치 내보내기, 학습 데이터 접근, 운영 구성 변경)이 다단계 인증 또는 세션 재검증이 포함된 단계별 인증을 요구하는지 확인하십시오. |  1  | D/V |
| 5.1.3 | 연합 AI 에이전트가 최대 24시간의 수명을 가지며 출처에 대한 암호학적 증명을 포함하는 서명된 JWT 어설션을 통해 인증하는지 확인하십시오.           |  3  | D/V |

---

## C5.2 권한 부여 및 정책

명시적인 권한 모델과 감사 기록을 포함하여 모든 AI 리소스에 대한 접근 제어를 구현하십시오.

|   #   | 설명                                                                                                         | 레벨  | 역할  |
| :---: | ---------------------------------------------------------------------------------------------------------- | :-: | :-: |
| 5.2.1 | 모든 AI 리소스(데이터셋, 모델, 엔드포인트, 벡터 컬렉션, 임베딩 인덱스, 컴퓨트 인스턴스)가 명시적인 허용 목록과 기본 거부 정책을 통해 역할 기반 접근 제어를 적용하는지 확인하십시오. |  1  | D/V |
| 5.2.2 | 모든 접근 제어 수정이 타임스탬프, 행위자 신원, 리소스 식별자 및 권한 변경과 함께 변경 불가능하게 기록되었는지 확인하십시오.                                    |  1  |  V  |
| 5.2.3 | 데이터 분류 레이블(PII, PHI, 독점 정보 등)이 파생된 리소스(임베딩, 프롬프트 캐시, 모델 출력 등)에 자동으로 전파되는지 확인하십시오.                          |  2  |  D  |
| 5.2.4 | 무단 접근 시도 및 권한 상승 이벤트가 상황별 메타데이터와 함께 실시간 경고를 발생시키는지 확인하십시오.                                                 |  2  | D/V |
| 5.2.5 | 인가 결정이 전용 정책 엔진(OPA, Cedar 또는 이에 상응하는 엔진)으로 외부화되었는지 확인하십시오.                                                |  1  | D/V |
| 5.2.6 | 정책이 사용자 역할 또는 그룹, 리소스 분류, 요청 컨텍스트, 테넌트 격리 및 시간적 제약과 같은 동적 속성을 런타임에 평가하는지 확인하십시오.                           |  1  | D/V |
| 5.2.7 | 정책 캐시의 TTL(수명)이 고감도 자원에 대해 5분을 초과하지 않고, 캐시 무효화 기능이 있는 표준 자원에 대해 1시간을 초과하지 않는지 확인하십시오.                      |  3  | D/V |

## C5.3 쿼리 시점 보안 적용

필수 필터링 및 행 수준 보안 정책을 사용하여 데이터베이스 계층 보안 제어를 구현하십시오.

|   #   | 설명                                                                                            | 레벨  | 역할  |
| :---: | --------------------------------------------------------------------------------------------- | :-: | :-: |
| 5.3.1 | 모든 벡터 데이터베이스 및 SQL 쿼리에 데이터베이스 엔진 수준에서 강제되는 필수 보안 필터(테넌트 ID, 민감도 라벨, 사용자 범위)가 포함되어 있는지 검증하십시오. |  1  | D/V |
| 5.3.2 | 모든 벡터 데이터베이스, 검색 인덱스 및 학습 데이터셋에 대해 행 수준 보안 정책과 필드 수준 마스킹이 정책 상속과 함께 활성화되어 있는지 검증하십시오.         |  1  | D/V |
| 5.3.3 | 인증 실패 평가가 즉시 쿼리를 중단하고 명확한 인증 오류 코드를 반환하는지 확인하십시오.                                             |  2  |  D  |
| 5.3.4 | 쿼리 재시도 메커니즘이 활성 사용자 세션 내에서 동적인 권한 변경을 반영하기 위해 권한 부여 정책을 재평가하는지 확인하십시오.                        |  3  | D/V |

## C5.4 출력 필터링 및 데이터 손실 방지

AI 생성 콘텐츠에서 무단 데이터 노출을 방지하기 위해 후처리 제어를 배포하십시오.

|   #   | 설명                                                                                | 레벨  | 역할  |
| :---: | --------------------------------------------------------------------------------- | :-: | :-: |
| 5.4.1 | 사후 추론 필터링 메커니즘이 요청자에게 콘텐츠를 전달하기 전에 무단 PII, 기밀 정보 및 독점 데이터를 검색하고 수정하는지 확인하십시오.     |  1  | D/V |
| 5.4.2 | 모델 출력에서 인용, 참고문헌 및 출처 표기가 호출자의 권한과 일치하는지 검증하고, 권한이 없는 접근이 감지되면 이를 제거하십시오.         |  1  | D/V |
| 5.4.3 | 사용자 권한 수준 및 데이터 분류에 따라 출력 형식 제한(정제된 PDF, 메타데이터가 제거된 이미지, 승인된 파일 유형)이 적용되는지 확인합니다. |  2  |  D  |

## C5.5 다중 테넌트 격리

공유 AI 인프라에서 테넌트 간 암호화 및 논리적 격리를 보장하십시오.

|   #   | 설명                                                                                          | 레벨  | 역할  |
| :---: | ------------------------------------------------------------------------------------------- | :-: | :-: |
| 5.5.1 | 메모리 공간, 임베딩 저장소, 캐시 항목 및 임시 파일이 테넌트별로 네임스페이스 구분되어 있으며, 테넌트 삭제 또는 세션 종료 시 안전하게 삭제되는지 확인하십시오. |  1  | D/V |
| 5.5.2 | 모든 API 요청에 대해 세션 컨텍스트와 사용자 권한에 대해 암호학적으로 검증된 인증된 테넌트 식별자가 포함되어 있는지 확인하십시오.                  |  1  | D/V |
| 5.5.3 | 서비스 메시 및 컨테이너 오케스트레이션 플랫폼 내에서 테넌트 간 통신에 대해 네트워크 정책이 기본 거부 규칙을 구현하는지 확인하십시오.                 |  2  |  D  |
| 5.5.4 | 고객 관리 키(CMK) 지원을 통해 테넌트별로 암호화 키가 고유하며 테넌트 데이터 저장소 간에 암호화 격리가 이루어지는지 검증하십시오.                 |  3  |  D  |

## C5.6 자율 에이전트 권한 부여

범위가 지정된 권한 토큰과 지속적인 승인을 통해 AI 에이전트 및 자율 시스템의 제어 권한을 관리합니다.

|   #   | 설명                                                                                   | 레벨  | 역할  |
| :---: | ------------------------------------------------------------------------------------ | :-: | :-: |
| 5.6.1 | 자율 에이전트가 허용된 작업, 접근 가능한 리소스, 시간 제한 및 운영 제약 조건을 명확하게 나열한 범위 지정 권한 토큰을 받는지 확인하세요.      |  1  | D/V |
| 5.6.2 | 고위험 기능(파일 시스템 접근, 코드 실행, 외부 API 호출, 금융 거래)이 기본적으로 비활성화되어 있으며 명확한 권한 부여가 필요함을 확인하십시오. |  1  | D/V |
| 5.6.3 | 기능 토큰이 사용자 세션에 바인딩되어 있으며, 암호화 무결성 보호를 포함하고, 오프라인 시나리오에서 저장되거나 재사용되지 않도록 확인하십시오.      |  2  |  D  |
| 5.6.4 | 에이전트가 시작한 작업이 ABAC 정책 엔진을 통해 권한 부여를 받았는지 확인합니다.                                      |  2  |  V  |
