# 9 자율 오케스트레이션 및 에이전트 행위 보안

## 제어 목표

자율형 또는 다중 에이전트 AI 시스템이 명시적으로 의도되고 인증되며 감사 가능하고 제한된 비용 및 위험 한도 내에서만 동작할 수 있도록 보장합니다. 이는 자율 시스템 침해, 도구 오용, 에이전트 루프 탐지, 통신 탈취, 신원 위조, 군집 조작, 의도 조작과 같은 위협으로부터 보호합니다.

## 9.10 에이전트 추론 전략 보안

ReAct, Chain-of-Thought, Tree-of-Thoughts 접근 방식을 포함한 다양한 추론 전략의 안전한 선택 및 실행.

|   #    | 설명                                                                                                             | 레벨  | 역할  |
| :----: | -------------------------------------------------------------------------------------------------------------- | :-: | :-: |
| 9.10.1 | 추론 전략 선택이 결정론적 기준(입력 복잡성, 작업 유형, 보안 컨텍스트)을 사용하는지 확인하고 동일한 보안 컨텍스트 내에서 동일한 입력이 동일한 전략 선택을 생성하는지 검증합니다.          |  1  | D/V |
| 9.10.2 | 각 추론 전략(ReAct, Chain-of-Thought, Tree-of-Thoughts)이 그 인지적 접근 방식에 맞는 전용 입력 검증, 출력 정제, 실행 시간 제한을 갖추고 있는지 확인하십시오. |  1  | D/V |
| 9.10.3 | 이성 전략 전환이 입력 특성, 선택 기준 값 및 실행 메타데이터를 포함한 완전한 컨텍스트와 함께 감사 추적 재구성을 위해 기록되는지 확인하십시오.                              |  2  | D/V |
| 9.10.4 | Tree-of-Thoughts 추론이 정책 위반, 자원 한도, 또는 안전 경계가 감지될 때 탐색을 종료하는 분기 가지치기 메커니즘을 포함하는지 확인하십시오.                        |  2  | D/V |
| 9.10.5 | ReAct (Reason-Act-Observe) 사이클에는 각 단계에서 검증 체크포인트가 포함되어 있는지 확인하십시오: 추론 단계 검증, 행동 승인, 그리고 진행하기 전 관찰 데이터 정제.      |  2  | D/V |
| 9.10.6 | 추론 전략 성능 지표(실행 시간, 자원 사용량, 출력 품질)가 구성된 임계값을 벗어날 때 자동 경고와 함께 모니터링되는지 확인하십시오.                                    |  3  | D/V |
| 9.10.7 | 여러 전략을 결합하는 하이브리드 추론 방식이 모든 구성 전략의 입력 검증과 출력 제약을 유지하며 어떠한 보안 통제도 우회하지 않는지 확인하십시오.                              |  3  | D/V |
| 9.10.8 | 추론 전략 보안 테스트에는 잘못된 입력을 사용한 퍼징, 전략 전환을 강제하기 위해 고안된 적대적 프롬프트, 그리고 각 인지 접근법에 대한 경계 조건 테스트가 포함되는지 확인하십시오.          |  3  | D/V |

| 9.11.2 | 에이전트 상태 전환이 암호학적으로 서명되고, 타임스탬프가 찍히며, 트리거 이벤트, 이전 상태 해시, 새로운 상태 해시, 수행된 보안 검증을 포함한 완전한 컨텍스트와 함께 기록되었는지 확인하십시오.    |  2  | D/V |
| 9.11.3 | 에이전트 종료 절차에 암호화 삭제 또는 다중 통과 덮어쓰기를 통한 안전한 메모리 삭제, 인증서 기관 알림을 포함한 자격 증명 폐기, 그리고 변조 방지 종료 인증서 생성을 포함하는지 확인하십시오.     |  2  | D/V |

## C9.13 모델 컨텍스트 프로토콜(MCP) 보안

컨텍스트 혼동, 무단 도구 호출 또는 테넌트 간 데이터 노출을 방지하기 위해 MCP 기반 도구 및 리소스 통합의 안전한 검색, 인증, 권한 부여, 전송 및 사용을 보장합니다.

### 구성 요소 무결성 및 공급망 위생

|   #    | 설명                                                                                                               | 레벨  | 역할  |
| :----: | ---------------------------------------------------------------------------------------------------------------- | :-: | :-: |
| 9.13.1 | MCP 서버, 클라이언트 및 도구 구현이 보안에 취약한 함수 노출, 안전하지 않은 기본값, 인증 누락 또는 입력 검증 누락을 식별하기 위해 수동 검토되거나 자동 분석되는지 확인합니다.           |  1  | D/V |
| 9.13.2 | 외부 또는 오픈소스 MCP 서버나 패키지가 통합 전에 자동화된 취약점 및 공급망 스캔(예: SCA)을 거치도록 확인하고, 알려진 심각 취약점을 가진 구성 요소는 사용하지 않도록 합니다.          |  1  | D/V |
| 9.13.3 | MCP 서버 및 클라이언트 구성 요소가 신뢰할 수 있는 출처에서만 획득되었는지 확인하고, 서명, 체크섬 또는 안전한 패키지 메타데이터를 사용하여 검증하며, 변조되었거나 서명되지 않은 빌드는 거부합니다. |  1  | D/V |

### 인증 및 권한 부여

|   #    | 설명                                                                                                                  | 레벨  | 역할  |
| :----: | ------------------------------------------------------------------------------------------------------------------- | :-: | :-: |
| 9.13.4 | MCP 클라이언트와 서버가 강력한 비사용자 자격 증명(예: mTLS, 서명된 토큰 또는 플랫폼 발급 식별자)을 사용하여 상호 인증하는지 확인하고, 인증되지 않은 MCP 엔드포인트가 거부되는지 검증합니다.   |  2  | D/V |
| 9.13.5 | MCP 서버가 명시적인 소유자, 환경 및 리소스 정의가 필요한 통제된 기술 온보딩 메커니즘을 통해 등록되었는지 확인하십시오; 등록되지 않았거나 탐지할 수 없는 서버는 운영 환경에서 호출할 수 없어야 합니다. |  2  | D/V |
| 9.13.6 | 각 MCP 도구 또는 리소스가 명확한 권한 범위(예: 읽기 전용, 제한된 쿼리, 부작용 수준)를 정의하고, 에이전트가 할당된 범위를 벗어난 MCP 기능을 호출할 수 없음을 확인합니다.              |  2  | D/V |

### 보안 전송 및 네트워크 경계 보호

|    #    | 설명                                                                                                                                               | 레벨  | 역할  |
| :-----: | ------------------------------------------------------------------------------------------------------------------------------------------------ | :-: | :-: |
| 9.13.7  | 인증되고 암호화된 스트리머블-HTTP가 프로덕션 환경에서 기본 MCP 전송 방식으로 사용되는지 확인하십시오; 대체 전송 방식(stdio, SSE)은 명확한 정당성 아래 로컬 또는 엄격하게 통제된 환경에만 제한됩니다.                         |  2  | D/V |
| 9.13.8  | 스트리머블-HTTP MCP 전송이 인증된 암호화 채널(TLS 1.3 이상)을 사용하며, 인증서 검증과 전달 비밀성을 통해 스트리밍된 MCP 메시지의 기밀성과 무결성을 보장하는지 확인하십시오.                                       |  2  | D/V |
| 9.13.9  | SSE 기반 MCP 전송이 사설 인증된 내부 채널 내에서만 사용되도록 확인하고 TLS, 인증, 스키마 검증, 페이로드 크기 제한 및 속도 제한을 강제 적용하십시오; SSE 엔드포인트는 공용 인터넷에 노출되어서는 안 됩니다.                     |  2  | D/V |
| 9.13.10 | MCP 서버가  검증하는지 확인하십시오 `Origin` 그리고`Host` 모든 HTTP 기반 전송(SSE 및 스트리밍 HTTP 포함)에 헤더를 추가하여 DNS 리바인딩 공격을 방지하고, 신뢰할 수 없거나 일치하지 않거나 누락된 출처로부터의 요청을 거부합니다. |  2  | D/V |
