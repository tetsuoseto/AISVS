# 부록 D: AI 지원 보안 코딩 거버넌스 및 검증

## 목표

이 장에서는 소프트웨어 개발 중 AI 지원 코딩 도구의 안전하고 효과적인 사용을 위한 기본 조직 통제를 정의하며, SDLC 전반에 걸쳐 보안성과 추적 가능성을 보장합니다.

---

## AD.1 AI 지원 보안 코딩 워크플로우

기존 보안 게이트를 약화시키지 않으면서 조직의 보안 소프트웨어 개발 수명 주기(SSDLC)에 AI 도구를 통합하십시오.

|   #    | 설명                                                                    | 레벨  | 역할  |
| :----: | --------------------------------------------------------------------- | :-: | :-: |
| AD.1.1 | 문서화된 워크플로우가 AI 도구가 언제 어떻게 코드를 생성, 리팩터링 또는 검토할 수 있는지 설명하는지 확인하십시오.     |  1  | D/V |
| AD.1.2 | 워크플로우가 각 SSDLC 단계(설계, 구현, 코드 리뷰, 테스트, 배포)에 매핑되는지 확인하십시오.              |  2  |  D  |
| AD.1.3 | AI가 생성한 코드에 대해 메트릭(예: 취약점 밀도, 평균 탐지 시간)이 수집되고 인간만의 기준선과 비교되는지 확인하십시오. |  3  | D/V |

---

## AD.2 AI 도구 자격 인증 및 위협 모델링

AI 코딩 도구는 도입 전에 보안 기능, 위험 및 공급망 영향에 대해 평가되어야 합니다.

|   #    | 설명                                                                            | 레벨  | 역할  |
| :----: | ----------------------------------------------------------------------------- | :-: | :-: |
| AD.2.1 | 각 AI 도구에 대한 위협 모델이 오용, 모델 역전, 데이터 누출 및 종속성 체인 위험을 식별하는지 검증합니다.                |  1  | D/V |
| AD.2.2 | 도구 평가에 지역 구성 요소의 정적/동적 분석과 SaaS 엔드포인트(TLS, 인증/권한 부여, 로깅) 평가가 포함되어 있는지 확인하십시오. |  2  |  D  |
| AD.2.3 | 평가가 인정된 프레임워크를 따른다는 것을 확인하고 주요 버전 변경 후에 평가가 다시 수행되는지 확인하십시오.                  |  3  | D/V |

---

## AD.3 안전한 프롬프트 및 컨텍스트 관리

AI 모델을 위한 프롬프트나 컨텍스트를 구성할 때 비밀, 독점 코드 및 개인 데이터의 유출을 방지하십시오.

|   #    | 설명                                                                  | 레벨  | 역할  |
| :----: | ------------------------------------------------------------------- | :-: | :-: |
| AD.3.1 | 작성된 지침이 프롬프트에 비밀 정보, 자격 증명 또는 기밀 데이터를 보내는 것을 금지하는지 확인하십시오.          |  1  | D/V |
| AD.3.2 | 기술 통제(클라이언트 측 삭제, 승인된 컨텍스트 필터)가 민감한 데이터 조각을 자동으로 제거하는지 확인하십시오.      |  2  |  D  |
| AD.3.3 | 프롬프트와 응답이 토큰화되고 전송 중 및 저장 시 암호화되었으며 보존 기간이 데이터 분류 정책을 준수하는지 확인하십시오. |  3  | D/V |

---

## AD.4 AI 생성 코드 검증

코드가 병합되거나 배포되기 전에 AI 출력으로 인해 도입된 취약점을 감지하고 수정하십시오.

|   #    | 설명                                                                                      | 레벨  | 역할  |
| :----: | --------------------------------------------------------------------------------------- | :-: | :-: |
| AD.4.1 | AI로 생성된 코드는 항상 사람의 코드 리뷰를 거치는지 확인하세요.                                                   |  1  | D/V |
| AD.4.2 | AI 생성 코드를 포함하는 모든 풀 리퀘스트에 대해 자동 스캐너(SAST/IAST/DAST)를 실행하고 심각한 문제 발견 시 병합을 차단하는지 확인하십시오. |  2  |  D  |
| AD.4.3 | 차등 퍼징 테스트 또는 속성 기반 테스트가 보안에 중요한 동작(예: 입력 검증, 권한 부여 로직)을 증명하는지 확인하십시오.                   |  3  | D/V |

---

## AD.5 코드 제안의 설명 가능성 및 추적 가능성

감사자와 개발자에게 제안이 왜 제시되었고 어떻게 발전되었는지에 대한 통찰을 제공합니다.

|   #    | 설명                                                                           | 레벨  | 역할  |
| :----: | ---------------------------------------------------------------------------- | :-: | :-: |
| AD.5.1 | 프롬프트/응답 쌍이 커밋 ID와 함께 기록되었는지 확인하십시오.                                          |  1  | D/V |
| AD.5.2 | 개발자가 제안을 지원하는 모델 인용문(학습 스니펫, 문서)을 표시할 수 있는지 확인합니다.                           |  2  |  D  |
| AD.5.3 | 설명 가능성 보고서가 설계 산출물과 함께 저장되고 보안 검토에서 참조되어 ISO/IEC 42001 추적성 원칙을 충족하는지 확인하십시오. |  3  | D/V |

---

## AD.6 연속 피드백 및 모델 미세 조정

부정적인 드리프트를 방지하면서 시간이 지남에 따라 모델 보안 성능을 향상시킵니다.

|   #    | 설명                                                                               | 레벨  | 역할  |
| :----: | -------------------------------------------------------------------------------- | :-: | :-: |
| AD.6.1 | 개발자가 안전하지 않거나 비준수인 제안을 표시할 수 있고, 이러한 표시가 추적되는지 확인합니다.                            |  1  | D/V |
| AD.6.2 | 집계된 피드백이 검증된 보안 코딩 코퍼스(예: OWASP 치트 시트)를 활용한 주기적인 파인튜닝 또는 검색 강화 생성에 반영되는지 확인하십시오. |  2  |  D  |
| AD.6.3 | 폐쇄형 평가 하니스가 미세 조정 후마다 회귀 테스트를 실행하는지 확인하십시오; 보안 지표는 배포 전에 이전 기준을 충족하거나 초과해야 합니다.  |  3  | D/V |
