# C3 모델 생명주기 관리 및 변경 제어

## 제어 목표

AI 시스템은 무단 또는 안전하지 않은 모델 변경이 프로덕션에 반영되지 않도록 변경 관리 프로세스를 구현해야 합니다. 이 제어는 개발부터 배포 및 사용 중단까지 전체 수명 주기 동안 모델의 무결성을 보장하며, 이를 통해 신속한 사고 대응이 가능하고 모든 변경 사항에 대한 책임을 유지할 수 있습니다.

핵심 보안 목표: 무결성, 추적성 및 복구 가능성을 유지하는 통제된 프로세스를 통해 승인되고 검증된 모델만 프로덕션에 도달하도록 하는 것입니다.

---

## C3.1 모델 승인 및 무결성

검증된 무결성을 가진 권한 있는 모델만이 프로덕션 환경에 도달합니다.

|   #   | 설명                                                                                        | 수준  | 역할  |
| :---: | ----------------------------------------------------------------------------------------- | :-: | :-: |
| 3.1.1 | 배포 전에 모든 모델 아티팩트(가중치, 구성, 토크나이저)가 권한 있는 주체에 의해 암호화 서명되었는지 확인하십시오.                         |  1  | D/V |
| 3.1.2 | 의존성 추적이 모든 소비 시스템을 식별할 수 있도록 실시간 재고를 유지하는지 확인하십시오.                                        |  2  |  V  |
| 3.1.3 | 모델 출처 기록에 권한 부여 기관의 신원, 학습 데이터 체크섬, 합격/불합격 상태가 포함된 검증 테스트 결과 및 생성 타임스탬프가 포함되어 있는지 확인하십시오. |  3  | D/V |

---

## C3.2 모델 검증 및 테스트

모델은 배포 전에 정의된 보안 및 안전 검증을 통과해야 합니다.

|   #   | 설명                                                                                                     | 수준  | 역할  |
| :---: | ------------------------------------------------------------------------------------------------------ | :-: | :-: |
| 3.2.1 | 모델이 배포 전에 사전에 합의된 조직의 합격/불합격 기준을 포함하여 입력 검증, 출력 정화, 안전성 평가를 포함한 자동화된 보안 테스트를 거치는지 확인하십시오.              |  1  | D/V |
| 3.2.2 | 모든 모델 변경 사항(배포, 구성, 폐기)이 타임스탬프, 인증된 행위자 신원, 변경 유형 및 변경 전/후 상태를 포함하는 불변의 감사 기록을 생성하는지 확인하십시오.           |  1  |  V  |
| 3.2.3 | 사전에 지정된 권한 있는 담당자가 명시적 승인을 통해 오버라이드를 한 후에도, 문서화된 비즈니스 정당성이 있을 경우에 한하여 검증 실패가 모델 배포를 자동으로 차단하는지 확인하십시오. |  2  | D/V |

---

## C3.3 통제된 배포 및 롤백

모델 배포는 제어되고, 모니터링되며, 되돌릴 수 있어야 합니다.

|   #   | 설명                                                                                                       | 수준  | 역할  |
| :---: | -------------------------------------------------------------------------------------------------------- | :-: | :-: |
| 3.3.1 | 배포 프로세스가 모델 활성화 전에 암호화 서명을 검증하고 무결성 체크섬을 계산하여 불일치가 있을 경우 배포를 실패하도록 확인하십시오.                               |  1  | D/V |
| 3.3.2 | 프로덕션 배포가 사전 합의된 오류율, 지연 임계값 또는 보안 경고 기준에 따라 자동 롤백 트리거가 포함된 점진적 롤아웃 메커니즘(카나리 배포, 블루-그린 배포)을 구현하는지 확인하십시오. |  1  |  D  |
| 3.3.3 | 롤백 기능이 모델 상태(가중치, 구성, 종속성)를 원자적으로 완전히 복원하는지 확인하십시오.                                                      |  2  | D/V |
| 3.3.4 | 비상 모델 종료 기능이 사전 정의된 응답 시간 내에 모델 엔드포인트를 비활성화할 수 있는지 확인합니다.                                                |  3  | D/V |

---

## C3.4 보안 개발 관행

모델 개발 및 훈련 과정은 보안을 유지하기 위해 안전한 절차를 따라야 합니다.

|   #   | 설명                                                                                                     | 수준  | 역할  |
| :---: | ------------------------------------------------------------------------------------------------------ | :-: | :-: |
| 3.4.1 | 모델 개발, 테스트 및 운영 환경이 물리적 또는 논리적으로 분리되어 있는지 확인합니다. 이들은 공유 인프라가 없고, 별도의 접근 제어 및 격리된 데이터 저장소를 갖추고 있어야 합니다. |  1  | D/V |
| 3.4.2 | 모델 개발 산출물(하이퍼파라미터, 학습 스크립트, 구성 파일, 프롬프트 템플릿 등)이 버전 관리 시스템에 저장되고 학습에 사용되기 전에 동료 검토 승인이 필요함을 확인하십시오.     |  1  |  D  |
| 3.4.3 | 모델 훈련과 미세 조정이 네트워크 접근이 통제된 격리된 환경에서 이루어지는지 확인하십시오.                                                     |  2  | D/V |
| 3.4.4 | 모델 개발에 사용되기 전에 훈련 데이터 소스가 무결성 검사를 통해 검증되고, 문서화된 관리 기록이 있는 신뢰할 수 있는 출처를 통해 인증되었는지 확인하십시오.               |  2  |  D  |

## C3.5 모델 은퇴 및 폐기

모델은 더 이상 필요하지 않거나 보안 문제가 발견될 때 안전하게 폐기되어야 합니다.

|   #   | 설명                                                                    | 수준  | 역할  |
| :---: | --------------------------------------------------------------------- | :-: | :-: |
| 3.5.1 | 퇴역된 모델 아티팩트가 안전한 암호학적 소거를 사용하여 안전하게 삭제되었는지 확인하십시오.                    |  1  | D/V |
| 3.5.2 | 모델 은퇴 이벤트가 타임스탬프와 행위자 신원과 함께 기록되고, 모델 서명이 재사용을 방지하기 위해 취소되었는지 확인하십시오. |  2  |  V  |
