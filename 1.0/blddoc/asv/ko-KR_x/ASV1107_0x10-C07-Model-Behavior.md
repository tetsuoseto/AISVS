# C7 모델 동작, 출력 제어 및 안전 보증

## 제어 목표

모델 출력은 구조화되고, 신뢰할 수 있으며, 안전하고, 설명 가능하며, 생산 환경에서 지속적으로 모니터링되어야 합니다. 이렇게 하면 환각 현상, 개인정보 유출, 유해한 콘텐츠, 통제 불가능한 동작이 감소하고 사용자 신뢰 및 규제 준수가 향상됩니다.

---

## C7.1 출력 형식 준수

엄격한 스키마, 제한된 디코딩, 그리고 하류 검증은 잘못 형성되었거나 악의적인 콘텐츠가 확산되기 전에 차단합니다.

|   #   | 설명                                                                                         | 수준  | 역할  |
| :---: | ------------------------------------------------------------------------------------------ | :-: | :-: |
| 7.1.1 | 시스템 프롬프트에 응답 스키마(예: JSON 스키마)가 제공되어 있고, 모든 출력이 자동으로 검증되며, 비준수 출력은 수정 또는 거부가 이루어지는지 확인하십시오. |  1  | D/V |
| 7.1.2 | 오버플로우 또는 프롬프트 인젝션 사이드 채널을 방지하기 위해 제한된 디코딩(중지 토큰, 정규식, 최대 토큰 수)이 활성화되어 있는지 확인하십시오.          |  1  | D/V |
| 7.1.3 | 다운스트림 구성 요소가 출력을 신뢰할 수 없는 것으로 처리하고, 이를 스키마 또는 인젝션에 안전한 역직렬화기로 검증하는지 확인하십시오.                |  2  | D/V |
| 7.1.4 | 부적절한 출력 이벤트가 기록되고, 속도 제한되며, 모니터링에 표시되는지 확인하세요.                                             |  3  |  V  |

## C7.2 환각 감지 및 완화

불확실성 추정 및 폴백 전략은 조작된 답변을 억제합니다.

|   #   | 설명                                                                    | 수준  | 역할  |
| :---: | --------------------------------------------------------------------- | :-: | :-: |
| 7.2.1 | 토큰 수준 로그 확률, 앙상블 자기 일관성 또는 미세 조정된 환각 검출기가 각 답변에 신뢰도 점수를 할당하는지 확인하십시오. |  1  | D/V |
| 7.2.2 | 응답의 신뢰도 임계값 아래에서 대체 워크플로우(예: 검색 보강 생성, 보조 모델 또는 인간 검토)가 작동하는지 확인하십시오. |  1  | D/V |
| 7.2.3 | 환각 발생 사례가 근본 원인 메타데이터로 태그되어 사후 분석 및 미세 조정 파이프라인에 제공되는지 확인하십시오.        |  2  | D/V |
| 7.2.4 | 주요 모델 또는 지식 기반 업데이트 후 임계값과 감지기가 재보정되었는지 확인하십시오.                       |  3  | D/V |
| 7.2.5 | 대시보드 시각화가 환각률을 추적하는지 확인하십시오.                                          |  3  |  V  |

## C7.3 출력 안전 및 개인정보 보호 필터링

정책 필터와 레드팀 커버리지는 사용자와 기밀 데이터를 보호합니다.

|   #   | 설명                                                                       | 수준  | 역할  |
| :---: | ------------------------------------------------------------------------ | :-: | :-: |
| 7.3.1 | 사전 및 사후 생성 분류기가 정책에 부합하는 증오, 괴롭힘, 자해, 극단주의 및 성적으로 노골적인 내용을 차단하는지 확인하십시오. |  1  | D/V |
| 7.3.2 | PII/PCI 감지 및 자동 삭제가 모든 응답에서 실행되는지 확인하십시오; 위반 시 개인정보 보호 사건이 발생합니다.        |  1  | D/V |
| 7.3.3 | 기밀 태그(예: 영업 비밀)가 텍스트, 이미지 또는 코드에서 누출을 방지하기 위해 모달리티 전반에 걸쳐 전파되는지 확인하십시오.  |  2  |  D  |
| 7.3.4 | 필터 우회 시도나 고위험 분류에 대해 2차 승인 또는 사용자 재인증이 필요함을 확인하십시오.                      |  3  | D/V |
| 7.3.5 | 필터링 임계값이 법적 관할 구역 및 사용자 연령/역할 컨텍스트를 반영하는지 확인하십시오.                        |  3  | D/V |

## C7.4 출력 및 동작 제한

비율 제한과 승인 게이트는 악용과 과도한 자율성을 방지합니다.

|   #   | 설명                                                                                 | 수준  | 역할  |
| :---: | ---------------------------------------------------------------------------------- | :-: | :-: |
| 7.4.1 | 429 오류 발생 시 지수적 백오프로 요청, 토큰, 비용이 사용자별 및 API 키별 할당량에 따라 제한되는지 확인합니다.                |  1  |  D  |
| 7.4.2 | 권한이 필요한 작업(파일 쓰기, 코드 실행, 네트워크 호출)이 정책 기반 승인 또는 인간 검토 과정이 필요함을 확인하십시오.              |  1  | D/V |
| 7.4.3 | 교차 모달 일관성 검사가 동일한 요청에 대해 생성된 이미지, 코드 및 텍스트가 악성 콘텐츠를 은닉하는 데 사용될 수 없음을 보장하는지 확인하십시오. |  2  | D/V |
| 7.4.4 | 에이전트 위임 깊이, 재귀 한도 및 허용된 도구 목록이 명확하게 설정되었는지 확인하십시오.                                 |  2  |  D  |
| 7.4.5 | 제한 위반 시 SIEM 수집을 위한 구조화된 보안 이벤트가 발생하는지 확인하십시오.                                     |  3  |  V  |

## C7.5 출력 설명 가능성

투명한 신호는 사용자 신뢰와 내부 디버깅을 향상시킵니다.

|   #   | 설명                                                              | 수준  | 역할  |
| :---: | --------------------------------------------------------------- | :-: | :-: |
| 7.5.1 | 위험 평가에서 적절하다고 판단될 때 사용자에게 표시되는 신뢰 점수 또는 간략한 추론 요약이 노출되는지 확인합니다. |  2  | D/V |
| 7.5.2 | 생성된 설명이 민감한 시스템 프롬프트나 독점 데이터를 노출하지 않는지 확인하십시오.                  |  2  | D/V |
| 7.5.3 | 시스템이 토큰 수준의 로그 확률 또는 어텐션 맵을 캡처하여 승인된 검사를 위해 저장하는지 확인하십시오.       |  3  |  D  |
| 7.5.4 | 설명 가능성 산출물이 감사 가능성을 위해 모델 릴리스와 함께 버전 관리되는지 확인하십시오.              |  3  |  V  |

## C7.6 모니터링 통합

실시간 관찰성은 개발과 운영 간의 연결 고리를 완성합니다.

|   #   | 설명                                                                     | 수준  | 역할  |
| :---: | ---------------------------------------------------------------------- | :-: | :-: |
| 7.6.1 | 메트릭(스키마 위반, 환각률, 독성, PII 유출, 지연 시간, 비용)이 중앙 모니터링 플랫폼으로 스트리밍되는지 확인하십시오. |  1  |  D  |
| 7.6.2 | 각 안전 지표에 대해 경고 임계값이 정의되어 있으며, 대기 중인 담당자에게 에스컬레이션 경로가 있는지 확인하십시오.       |  1  |  V  |
| 7.6.3 | 대시보드가 출력 이상 현상을 모델/버전, 기능 플래그, 그리고 상위 데이터 변경 사항과 연관시키는지 확인하십시오.        |  2  |  V  |
| 7.6.4 | 문서화된 MLOps 워크플로 내에서 모니터링 데이터가 재훈련, 미세 조정 또는 규칙 업데이트로 피드백되는지 확인하십시오.    |  2  | D/V |
| 7.6.5 | 민감한 로그 유출을 방지하기 위해 모니터링 파이프라인이 침투 테스트되고 접근 제어되어 있는지 확인하십시오.            |  3  |  V  |

## 7.7 생성 미디어 안전장치

AI 시스템이 정책 제약, 출력 검증 및 추적 가능성을 통해 불법적이거나 유해하거나 무단의 미디어 콘텐츠를 생성하지 않도록 보장합니다.

|   #   | 설명                                                                                          | 수준  | 역할  |
| :---: | ------------------------------------------------------------------------------------------- | :-: | :-: |
| 7.7.1 | 시스템 프롬프트와 사용자 지침이 불법적이거나 유해한, 또는 비동의에 의한 딥페이크 미디어(예: 이미지, 비디오, 오디오) 생성을 명시적으로 금지하는지 확인하십시오. |  1  | D/V |
| 7.7.2 | 프롬프트가 사칭 시도, 성적으로 노골적인 딥페이크, 또는 동의 없이 실제 개인을 묘사하는 미디어 생성을 차단하도록 필터링되었는지 확인하십시오.             |  2  | D/V |
| 7.7.3 | 시스템이 무단 복제를 방지하기 위해 지각 해싱, 워터마크 감지 또는 지문 인식을 사용하는지 확인하십시오.                                  |  2  |  V  |
| 7.7.4 | 모든 생성된 미디어가 다운스트림 추적 가능성을 위해 암호학적으로 서명되거나, 워터마크가 삽입되었거나, 변조 방지 출처 메타데이터가 내장되었는지 확인하십시오.     |  3  | D/V |
| 7.7.5 | 우회 시도(예: 프롬프트 난독화, 속어, 적대적 문구)가 감지, 기록 및 속도 제한되는지 확인하고, 반복적인 악용 사례는 모니터링 시스템에 보고되는지 확인하십시오. |  3  |  V  |
