# AI 구성 요소 및 사용자에 대한 C5 접근 제어 및 신원 관리

## 제어 목표

AI 시스템을 위한 효과적인 접근 제어는 강력한 신원 관리, 상황 인식 권한 부여, 그리고 제로 트러스트 원칙에 따른 런타임 실행을 필요로 합니다. 이러한 제어는 인간, 서비스, 자율 에이전트가 명시적으로 부여된 범위 내에서만 모델, 데이터, 그리고 계산 자원과 상호작용하도록 하며, 연속적인 검증 및 감사 기능을 보장합니다.

---

## C5.1 신원 관리 및 인증

다단계 인증을 통해 모든 엔터티에 대해 암호학적으로 지원되는 신원을 설정하십시오.

|   #   | 설명                                                                                           | 수준  | 역할  |
| :---: | -------------------------------------------------------------------------------------------- | :-: | :-: |
| 5.1.1 | 모든 인간 사용자 및 서비스 주체가 OIDC 및/또는 SAML 프로토콜을 사용하여 중앙 집중식 엔터프라이즈 아이덴티티 제공자(IdP)를 통해 인증하는지 확인하십시오. |  1  | D/V |
| 5.1.2 | 고위험 작업(모델 배포, 가중치 내보내기, 학습 데이터 접근, 운영 환경 구성 변경)이 다단계 인증 또는 세션 재검증이 포함된 단계별 인증을 요구하는지 확인하십시오. |  1  | D/V |
| 5.1.3 | 연합 AI 에이전트가 최대 수명 24시간의 서명된 JWT 어서션을 통해 인증하며, 여기에는 출처에 대한 암호학적 증명이 포함되어 있는지 확인하십시오.          |  3  | D/V |

---

## C5.2 권한 부여 및 정책

명시적인 권한 모델과 감사 추적을 포함하여 모든 AI 자원에 대한 접근 제어를 구현하십시오.

|   #   | 설명                                                                                                         | 수준  | 역할  |
| :---: | ---------------------------------------------------------------------------------------------------------- | :-: | :-: |
| 5.2.1 | 모든 AI 리소스(데이터셋, 모델, 엔드포인트, 벡터 컬렉션, 임베딩 인덱스, 컴퓨트 인스턴스)가 명시적인 허용 목록과 기본 거부 정책을 갖춘 역할 기반 접근 제어를 시행하는지 확인하십시오. |  1  | D/V |
| 5.2.2 | 모든 접근 제어 수정 사항이 타임스탬프, 행위자 신원, 자원 식별자, 권한 변경 내용과 함께 불변적으로 기록되었는지 확인하십시오.                                   |  1  |  V  |
| 5.2.3 | 데이터 분류 레이블(PII, PHI, 독점 정보 등)이 파생된 리소스(임베딩, 프롬프트 캐시, 모델 출력)로 자동 전파되는지 확인하십시오.                              |  2  |  D  |
| 5.2.4 | 무단 접근 시도 및 권한 상승 이벤트가 상황별 메타데이터와 함께 실시간 알림을 유발하는지 확인하십시오.                                                  |  2  | D/V |
| 5.2.5 | 권한 부여 결정이 전용 정책 엔진(OPA, Cedar 또는 이에 상응하는 엔진)으로 외부화되어 있는지 확인하십시오.                                           |  1  | D/V |
| 5.2.6 | 정책이 실행 시간에 사용자 역할 또는 그룹, 리소스 분류, 요청 컨텍스트, 테넌트 격리, 그리고 시간적 제약과 같은 동적 속성을 평가하는지 확인하십시오.                      |  1  | D/V |
| 5.2.7 | 정책 캐시 수명 시간(TTL) 값이 고감도 자원의 경우 5분을 초과하지 않고, 표준 자원의 경우 캐시 무효화 기능과 함께 1시간을 초과하지 않는지 확인하세요.                   |  3  | D/V |

## C5.3 쿼리 실행 시 보안 적용

데이터베이스 계층 보안 제어를 필수 필터링 및 행 수준 보안 정책과 함께 구현하십시오.

|   #   | 설명                                                                                     | 수준  | 역할  |
| :---: | -------------------------------------------------------------------------------------- | :-: | :-: |
| 5.3.1 | 모든 벡터 데이터베이스 및 SQL 쿼리에 필수 보안 필터(테넌트 ID, 민감도 라벨, 사용자 범위)가 데이터베이스 엔진 수준에서 적용되는지 확인하십시오.  |  1  | D/V |
| 5.3.2 | 모든 벡터 데이터베이스, 검색 인덱스, 학습 데이터셋에 대해 정책 상속이 적용된 행 수준 보안 정책 및 필드 수준 마스킹이 활성화되어 있는지 확인하십시오. |  1  | D/V |
| 5.3.3 | 실패한 권한 평가가 즉시 쿼리를 중단하고 명확한 권한 오류 코드를 반환하는지 확인하십시오.                                     |  2  |  D  |
| 5.3.4 | 쿼리 재시도 메커니즘이 활성 사용자 세션 내에서 동적 권한 변경을 반영하기 위해 권한 정책을 재평가하는지 확인하십시오.                     |  3  | D/V |

## C5.4 출력 필터링 및 데이터 손실 방지

AI가 생성한 콘텐츠에서 무단 데이터 노출을 방지하기 위해 사후 처리 제어를 배포하십시오.

|   #   | 설명                                                                               | 수준  | 역할  |
| :---: | -------------------------------------------------------------------------------- | :-: | :-: |
| 5.4.1 | 추론 후 필터링 메커니즘이 요청자에게 콘텐츠를 전달하기 전에 무단 PII, 분류된 정보 및 독점 데이터를 스캔하고 삭제하는지 확인하십시오.    |  1  | D/V |
| 5.4.2 | 모델 출력물의 인용문, 참고문헌 및 출처 표기가 호출자의 권한과 대조되어 검증되며, 무단 접근이 감지될 경우 제거되었는지 확인하십시오.      |  1  | D/V |
| 5.4.3 | 사용자 권한 수준 및 데이터 분류에 따라 출력 형식 제한(정제된 PDF, 메타데이터 제거된 이미지, 승인된 파일 유형)이 적용되는지 확인합니다. |  2  |  D  |

## C5.5 다중 테넌트 격리

공유 AI 인프라에서 테넌트 간의 암호화 및 논리적 분리를 보장합니다.

|   #   | 설명                                                                                          | 수준  | 역할  |
| :---: | ------------------------------------------------------------------------------------------- | :-: | :-: |
| 5.5.1 | 메모리 공간, 임베딩 저장소, 캐시 항목 및 임시 파일이 테넌트별로 네임스페이스 분리되어 있으며, 테넌트 삭제 또는 세션 종료 시 안전하게 삭제되는지 검증하십시오. |  1  | D/V |
| 5.5.2 | 모든 API 요청에 대해 세션 컨텍스트와 사용자 권한에 대해 암호학적으로 검증된 인증된 테넌트 식별자가 포함되어 있는지 확인하십시오.                  |  1  | D/V |
| 5.5.3 | 서비스 메시 및 컨테이너 오케스트레이션 플랫폼 내에서 테넌트 간 통신에 대해 기본 거부 규칙이 네트워크 정책에 구현되어 있는지 확인하십시오.              |  2  |  D  |
| 5.5.4 | 고객 관리 키(CMK) 지원과 테넌트 데이터 저장소 간의 암호학적 격리를 통해 테넌트별로 암호화 키가 고유한지 검증합니다.                        |  3  |  D  |

## C5.6 자율 에이전트 권한 부여

범위가 지정된 기능 토큰과 지속적인 권한 부여를 통해 AI 에이전트 및 자율 시스템의 제어 권한을 관리합니다.

|   #   | 설명                                                                                        | 수준  | 역할  |
| :---: | ----------------------------------------------------------------------------------------- | :-: | :-: |
| 5.6.1 | 자율 에이전트가 허용된 작업, 접근 가능한 리소스, 시간 경계 및 운영 제약 조건을 명시적으로 나열하는 범위 지정 권한 토큰을 수신하는지 확인합니다.       |  1  | D/V |
| 5.6.2 | 높은 위험도가 있는 기능(파일 시스템 접근, 코드 실행, 외부 API 호출, 금융 거래)이 기본적으로 비활성화되어 있으며 명시적인 승인이 필요함을 확인하십시오. |  1  | D/V |
| 5.6.3 | 기능 토큰이 사용자 세션에 바인딩되어 있고, 암호학적 무결성 보호가 포함되어 있으며, 오프라인 상황에서도 저장되거나 재사용될 수 없음을 확인하십시오.       |  2  |  D  |
| 5.6.4 | 에이전트가 시작한 작업이 ABAC 정책 엔진을 통해 권한 검증을 받는지 확인하십시오.                                           |  2  |  V  |
