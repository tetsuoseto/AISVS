## C4.17 영지식 인프라

민감한 정보를 노출하지 않고 프라이버시 보호 AI 검증 및 인증을 위해 영지식 증명 시스템을 구현합니다.

|   #    | 설명                                                                                       | 수준  | 역할  |
| :----: | ---------------------------------------------------------------------------------------- | :-: | :-: |
| 4.17.1 | 제로 지식 증명(ZK-SNARKs, ZK-STARKs)이 모델 가중치나 학습 데이터를 노출하지 않고 AI 모델의 무결성과 학습 출처를 검증하는지 확인하십시오. |  3  | D/V |
| 4.17.2 | ZK 기반 인증 시스템이 신원 관련 정보를 공개하지 않고 AI 서비스에 대한 프라이버시 보호 사용자 인증을 가능하게 하는지 검증하십시오.             |  3  | D/V |
| 4.17.3 | 사설 집합 교집합(PSI) 프로토콜이 개별 데이터셋을 노출하지 않으면서 연합 AI를 위한 안전한 데이터 매칭을 가능하게 하는지 검증하십시오.           |  3  | D/V |
| 4.17.4 | 제로 지식 기계 학습(ZKML) 시스템이 올바른 계산에 대한 암호학적 증명과 함께 검증 가능한 AI 추론을 가능하게 하는지 확인하십시오.             |  3  | D/V |
| 4.17.5 | ZK-롤업이 배치 검증과 감소된 계산 오버헤드를 통해 확장 가능하고 개인정보를 보호하는 AI 거래 처리를 제공함을 검증하세요.                   |  3  | D/V |

| 4.18.2 | 전력 분석 방어에는 노이즈 주입, 전원 라인 필터링, 랜덤화된 실행 패턴이 포함되어 있는지 확인하세요.                |  3  | D/V |
| 4.18.3 | 캐시 기반 사이드 채널 완화 기법은 정보 누출을 방지하기 위해 캐시 파티셔닝, 무작위화, 플러시 명령어를 사용함을 확인합니다.   |  3  | D/V |

| 4.18.5 | 마이크로아키텍처 측면 채널 방어가 투기 실행 제어 및 메모리 접근 패턴 은폐를 포함하는지 확인하십시오.                |  3  | D/V |
