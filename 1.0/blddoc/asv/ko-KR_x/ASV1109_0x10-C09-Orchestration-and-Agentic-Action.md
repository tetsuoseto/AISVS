# 9 자율 오케스트레이션 및 대리 행동 보안

## 제어 목표

자율 또는 다중 에이전트 AI 시스템이 명시적으로 의도되고 인증되며 감사 가능하고 제한된 비용 및 위험 임계값 내에서만 작업을 실행할 수 있도록 보장해야 합니다. 이는 자율 시스템 침해, 도구 오용, 에이전트 루프 감지, 통신 탈취, 신원 위조, 군집 조작 및 의도 조작과 같은 위협으로부터 보호합니다.

암호화된 감사 추적 및 정의된 복구 절차를 통해 에이전트의 안전한 초기화, 상태 전환 및 종료를 보장합니다.

|   #    | 설명                                                                                                              | 수준  | 역할  |
| :----: | --------------------------------------------------------------------------------------------------------------- | :-: | :-: |
| 9.11.1 | 에이전트 초기화가 하드웨어 기반 자격 증명을 통한 암호화 신원 구축과 에이전트 ID, 타임스탬프, 구성 해시, 초기화 매개변수를 포함하는 변조 불가능한 시작 감사 로그를 포함하는지 확인하십시오.    |  1  | D/V |
| 9.11.2 | 에이전트 상태 전환이 암호화 서명되고, 타임스탬프가 기록되며, 트리거 이벤트, 이전 상태 해시, 새로운 상태 해시 및 수행된 보안 검증을 포함한 완전한 컨텍스트와 함께 로그에 기록되었는지 확인합니다. |  2  | D/V |
| 9.11.3 | 에이전트 종료 절차에 암호화 삭제 또는 다중 덮어쓰기 방식을 이용한 안전한 메모리 삭제, 자격 증명 취소 및 인증서 기관 알림, 변조 방지 종료 증명서 생성이 포함되어 있는지 확인하십시오.       |  2  | D/V |
| 9.11.4 | 에이전트 복구 메커니즘이 암호화 체크섬(SHA-256 최소)을 사용하여 상태 무결성을 검증하고, 손상이 발견될 경우 자동 경고 및 수동 승인 요구와 함께 알려진 정상 상태로 롤백하는지 확인하십시오.  |  3  | D/V |
| 9.11.5 | 에이전트 지속성 메커니즘이 민감한 상태 데이터를 에이전트별 AES-256 키로 암호화하고, 구성 가능한 일정(최대 90일)으로 안전한 키 교체를 무중단 배포 방식으로 구현하는지 검증하십시오.      |  3  | D/V |

## 9.12 도구 통합 보안 프레임워크

정의된 위험 평가 및 승인 프로세스와 함께 동적 도구 로딩, 실행 및 결과 검증을 위한 보안 통제.

|   #    | 설명                                                                                                                          | 수준  | 역할  |
| :----: | --------------------------------------------------------------------------------------------------------------------------- | :-: | :-: |
| 9.12.1 | 도구 설명자에 필수 권한(읽기/쓰기/실행), 위험 수준(낮음/중간/높음), 자원 제한(CPU, 메모리, 네트워크), 및 도구 매니페스트에 문서화된 검증 요구 사항을 명시하는 보안 메타데이터가 포함되어 있는지 확인하십시오. |  1  | D/V |
| 9.12.2 | 도구 실행 결과가 타임아웃 제한 및 오류 처리 절차와 함께 통합되기 전에 예상 스키마(JSON 스키마, XML 스키마) 및 보안 정책(출력 정제, 데이터 분류)에 대해 검증되었는지 확인하십시오.                |  1  | D/V |
| 9.12.3 | 도구 상호작용 로그에 권한 사용, 데이터 접근 패턴, 실행 시간, 리소스 소비 및 반환 코드 등 상세한 보안 컨텍스트가 포함되어 있으며 SIEM 통합을 위한 구조화된 로깅이 이루어지고 있음을 검증하십시오.          |  2  | D/V |
| 9.12.4 | 동적 도구 로딩 메커니즘이 PKI 인프라를 사용하여 디지털 서명을 검증하고, 실행 전에 샌드박스 격리 및 권한 검증을 포함한 보안 로딩 프로토콜을 구현하는지 확인하십시오.                             |  2  | D/V |
| 9.12.5 | 도구 보안 평가가 새로운 버전에 대해 정적 분석, 동적 테스트, 보안 팀 검토를 포함한 필수 승인 게이트와 함께 자동으로 트리거되며, 문서화된 승인 기준과 SLA 요구 사항이 있는지 확인하십시오.               |  3  | D/V |
