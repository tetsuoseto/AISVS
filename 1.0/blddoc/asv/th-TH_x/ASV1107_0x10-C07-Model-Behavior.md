# พฤติกรรมของโมเดล C7 การควบคุมผลลัพธ์ และการรับประกันความปลอดภัย

## วัตถุประสงค์ในการควบคุม

ผลลัพธ์ของโมเดลต้องมีโครงสร้างที่ชัดเจน เชื่อถือได้ ปลอดภัย อธิบายได้ และมีการติดตามอย่างต่อเนื่องในระบบการผลิต การดำเนินการดังกล่าวช่วยลดการสร้างข้อมูลเท็จ รั่วไหลของข้อมูลส่วนบุคคล เนื้อหาที่เป็นอันตราย และการทำงานที่เกินขอบเขต ในขณะที่เพิ่มความเชื่อถือจากผู้ใช้และการปฏิบัติตามกฎระเบียบ

การประมาณค่าความไม่แน่นอนและกลยุทธ์สำรองช่วยลดคำตอบที่ถูกสร้างขึ้นใหม่

| :---: | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :---: | :---: |
| 7.2.1 | ตรวจสอบให้แน่ใจว่าความน่าจะเป็นของโทเค็นระดับคำ, การตรวจสอบความสอดคล้องด้วยตัวเองแบบรวมกลุ่ม (ensemble self-consistency), หรือเครื่องตรวจจับภาพหลอนที่ฝึกปรับแต่งแล้ว กำหนดคะแนนความมั่นใจให้กับแต่ละคำตอบ |   1   |  D/V  |
| 7.2.2 | ตรวจสอบให้แน่ใจว่าการตอบกลับที่มีความมั่นใจต่ำกว่าค่าธรณีที่ตั้งไว้จะกระตุ้นเวิร์กโฟลว์สำรอง (เช่น การสร้างโดยเพิ่มการดึงข้อมูล, โมเดลสำรอง, หรือการตรวจสอบโดยมนุษย์)                                      |   1   |  D/V  |
| 7.2.3 | ตรวจสอบให้แน่ใจว่าเหตุการณ์ฮัลลูซิเนชันได้รับการติดแท็กด้วยเมทาดาท้ารากเหง้าและถูกส่งไปยังระบบวิเคราะห์หลังเกิดเหตุและการปรับแต่งโมเดล                                                                     |   2   |  D/V  |
| 7.2.4 | ตรวจสอบให้แน่ใจว่าเกณฑ์และตัวตรวจจับได้รับการปรับเทียบใหม่หลังจากการอัปเดตโมเดลใหญ่หรือฐานความรู้                                                                                                          |   3   |  D/V  |
| 7.2.5 | ตรวจสอบให้แน่ใจว่าวิสัยทัศน์บนแดชบอร์ดติดตามอัตราการเกิดภาพหลอน                                                                                                                                            |   3   |   V   |
