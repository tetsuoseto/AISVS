# 10 对抗鲁棒性与隐私防御

## 控制目标

确保人工智能模型在面对规避、推断、提取或投毒攻击时保持可靠、保护隐私并具备抗滥用能力。

---

## 10.1 模型对齐与安全

防范有害或违反政策的输出。

|   #    | 描述                                             | 级别  | 角色  |
| :----: | ---------------------------------------------- | :-: | :-: |
| 10.1.1 | 验证对齐测试套件（红队提示、越狱探测、禁止内容）是否进行了版本控制，并在每次模型发布时运行。 |  1  | D/V |
| 10.1.2 | 验证拒绝和安全完成保护栏是否被执行。                             |  1  |  D  |
| 10.1.3 | 验证自动评估器是否测量有害内容率并标记超过设定阈值的回归。                  |  2  | D/V |
| 10.1.4 | 验证反越狱训练是否有文档记录且可复现。                            |  2  |  D  |
| 10.1.5 | 验证正式的政策合规性证明或认证监控是否涵盖关键领域。                     |  3  |  V  |

---

## 10.2 对抗样本加固

增强对篡改输入的抵抗力。稳健的对抗训练和基准评分是当前的最佳实践。

|   #    | 描述                                | 级别  | 角色  |
| :----: | --------------------------------- | :-: | :-: |
| 10.2.1 | 验证项目仓库是否包含带有可复现种子的对抗训练配置。         |  1  |  D  |
| 10.2.2 | 验证对抗样本检测是否在生产流水线中触发阻断警报。          |  2  | D/V |
| 10.2.4 | 验证认证的鲁棒性证明或区间界限证书是否至少涵盖了最关键的顶级类别。 |  3  |  V  |
| 10.2.5 | 验证回归测试使用自适应攻击以确认没有可测量的鲁棒性损失。      |  3  |  V  |

---

## 10.3 成员推断缓解

限制判断记录是否在训练数据中的能力。差分隐私和置信度分数掩蔽仍然是目前已知的最有效防御措施。

|   #    | 描述                                  | 级别  | 角色  |
| :----: | ----------------------------------- | :-: | :-: |
| 10.3.1 | 验证每次查询的熵正则化或温度缩放是否能够减少过度自信的预测。      |  1  |  D  |
| 10.3.2 | 验证训练过程中对敏感数据集采用了ε-界限差分隐私优化。         |  2  |  D  |
| 10.3.3 | 验证攻击模拟（影子模型或黑盒）在保留数据上的攻击AUC是否≤0.60。 |  2  |  V  |

---

## 10.4 模型反演抗性

防止重建私有属性。最近的调查强调输出截断和差分隐私保证作为实用防御措施。

|   #    | 描述                            | 级别  | 角色  |
| :----: | ----------------------------- | :-: | :-: |
| 10.4.1 | 确认敏感属性绝不被直接输出；在必要时，使用分段或单向变换。 |  1  |  D  |
| 10.4.2 | 验证查询速率限制是否限制来自同一主体的重复自适应查询。   |  1  | D/V |
| 10.4.3 | 验证模型是否经过保护隐私的噪声训练。            |  2  |  D  |

---

## 10.5 模型提取防御

检测并阻止未经授权的克隆。建议采用水印技术和查询模式分析。

|   #    | 描述                                                 | 级别  | 角色  |
| :----: | -------------------------------------------------- | :-: | :-: |
| 10.5.1 | 验证推理网关是否强制执行针对模型记忆阈值调整的全局和每个 API 密钥的速率限制。          |  1  |  D  |
| 10.5.2 | 验证查询熵和输入复数性统计是否为自动提取检测器提供数据。                       |  2  | D/V |
| 10.5.3 | 验证脆弱或概率水印是否可以在对疑似克隆进行不超过 1,000 次查询时，以 p < 0.01 证明。 |  2  |  V  |
| 10.5.4 | 验证水印密钥和触发集合是否存储在硬件安全模块中，并且每年轮换。                    |  3  |  D  |
| 10.5.5 | 验证提取警报事件是否包含违规查询，并且已与事件响应剧本集成。                     |  3  |  V  |

---

## 10.6 推理时的中毒数据检测

识别并中和带有后门或被投毒的输入。

|   #    | 描述                                 | 级别  | 角色  |
| :----: | ---------------------------------- | :-: | :-: |
| 10.6.1 | 验证输入在模型推理前通过异常检测器（例如，STRIP，一致性评分）。 |  1  |  D  |
| 10.6.2 | 验证检测器阈值是否在干净/中毒验证集上调优，以实现低于5%的误报率。 |  1  |  V  |
| 10.6.3 | 验证被标记为中毒的输入是否会触发软阻断和人工审核流程。        |  2  |  D  |
| 10.6.4 | 验证检测器是否通过自适应、无触发器的后门攻击进行了压力测试。     |  2  |  V  |
| 10.6.5 | 验证检测效能指标是否已记录，并定期使用最新的威胁情报重新评估。    |  3  |  D  |

---

## 10.7 动态安全策略适应

基于威胁情报和行为分析的实时安全策略更新。

|   #    | 描述                                   | 级别  | 角色  |
| :----: | ------------------------------------ | :-: | :-: |
| 10.7.1 | 验证安全策略能够在不重启代理的情况下动态更新，同时保持策略版本的完整性。 |  1  | D/V |
| 10.7.2 | 验证策略更新是否由授权的安全人员进行加密签名，并在应用之前进行验证。   |  2  | D/V |
| 10.7.3 | 验证动态策略更改是否记录了完整的审计追踪，包括理由、审批链和回滚程序。  |  2  | D/V |
| 10.7.4 | 验证自适应安全机制是否根据风险环境和行为模式调整威胁检测的灵敏度。    |  3  | D/V |
| 10.7.5 | 验证策略调整决策是否具有可解释性，并包含供安全团队审查的证据轨迹。    |  3  | D/V |

---

## 10.8 基于反射的安全分析

通过代理自我反思和元认知分析进行安全验证。

|   #    | 描述                                   | 级别  | 角色  |
| :----: | ------------------------------------ | :-: | :-: |
| 10.8.1 | 验证代理反思机制是否包含以安全为重点的决策和行动自我评估。        |  1  | D/V |
| 10.8.2 | 验证反射输出是否经过验证，以防止通过对抗性输入操纵自我评估机制。     |  2  | D/V |
| 10.8.3 | 验证元认知安全分析是否能识别代理推理过程中的潜在偏见、操控或妥协。    |  2  | D/V |
| 10.8.4 | 验证基于反射的安全警告是否触发了增强监控和潜在的人为干预流程。      |  3  | D/V |
| 10.8.5 | 验证从安全反思中持续学习能够提升威胁检测能力，同时不降低合法功能的性能。 |  3  | D/V |

---

## 10.9 演化与自我改进安全

具备自我修改和进化能力的代理系统的安全控制措施。

|   #    | 描述                            | 级别  | 角色  |
| :----: | ----------------------------- | :-: | :-: |
| 10.9.1 | 验证自我修改能力是否仅限于具有正式验证边界的指定安全区域。 |  1  | D/V |
| 10.9.2 | 确保在实施演进提案之前进行安全影响评估。          |  2  | D/V |
| 10.9.3 | 验证自我改进机制包括带有完整性验证的回滚功能。       |  2  | D/V |
| 10.9.4 | 验证元学习安全性是否防止了对改进算法的对抗性操纵。     |  3  | D/V |
| 10.9.5 | 验证递归自我改进受限于形式安全约束，并提供收敛的数学证明。 |  3  | D/V |
