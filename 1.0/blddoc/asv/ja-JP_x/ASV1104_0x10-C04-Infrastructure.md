# C4 インフラストラクチャ、構成および展開セキュリティ

## 制御目標

AIインフラストラクチャは、権限昇格、サプライチェーンの改ざん、横方向移動に対して、セキュアな構成、ランタイム分離、信頼されたデプロイメントパイプライン、および包括的な監視を通じて強固に保護されなければなりません。検証され承認されたインフラストラクチャコンポーネントのみが、セキュリティ、整合性、および監査可能性を確保する管理されたプロセスを経て本番環境に到達します。

コアセキュリティ目標：暗号的に署名され、脆弱性スキャンが実施されたインフラストラクチャコンポーネントのみが、セキュリティポリシーを強制し、不変の監査記録を維持する自動検証パイプラインを通じて本番環境に到達すること。

---

## C4.1 ランタイム環境の分離

OSレベルの分離プリミティブを通じて、コンテナの脱出および権限昇格を防止します。

|   #   | 説明                                                                                                                   | レベル | 役割  |
| :---: | -------------------------------------------------------------------------------------------------------------------- | :-: | :-: |
| 4.1.1 | すべてのAIワークロードが、例えばコンテナの場合に不要なLinuxの機能を削除するなどして、オペレーティングシステム上で必要最低限の権限で実行されていることを確認してください。                             |  1  | D/V |
| 4.1.2 | ワークロードがサンドボックス化、seccompプロファイル、AppArmor、SELinuxまたは同様の技術によって悪用を制限されて保護されており、かつその設定が適切であることを確認してください。                   |  1  | D/V |
| 4.1.3 | ワークロードが読み取り専用のルートファイルシステムで実行されていること、および書き込み可能なマウントポイントは明示的に定義され、制限付きオプション（例：noexec、nosuid、nodev）で強化されていることを検証してください。 |  2  | D/V |
| 4.1.4 | ランタイムモニタリングが特権昇格およびコンテナ脱出の動作を検出し、違反プロセスを自動的に終了することを確認してください。                                                         |  2  | D/V |
| 4.1.5 | 高リスクのAIワークロードが、リモート認証に成功した後にのみ、ハードウェア分離された環境（例：TEE、信頼されたハイパーバイザー、またはベアメタルノード）で実行されることを検証してください。                      |  3  | D/V |

## C4.2 セキュアビルド＆デプロイメントパイプライン

再現可能なビルドと署名されたアーティファクトを通じて、暗号学的な整合性とサプライチェーンの安全性を確保します。

|   #   | 説明                                                                             | レベル | 役割  |
| :---: | ------------------------------------------------------------------------------ | :-: | :-: |
| 4.2.1 | ビルドが再現可能であることを検証し、ビルド成果物に対して適切な署名付きのプロビナンスメタデータを生成し、それが独立して検証可能であることを確認してください。 |  1  | D/V |
| 4.2.2 | ビルドがソフトウェア部品表（SBOM）を生成し、展開前に署名されていることを確認してください。                                |  2  | D/V |
| 4.2.3 | ビルド成果物（例：コンテナイメージ）の署名および由来メタデータがデプロイ時に検証され、未検証の成果物が拒否されることを確認してください。           |  2  | D/V |

## C4.3 ネットワークセキュリティとアクセス制御

デフォルト拒否ポリシーと暗号化通信を使用してゼロトラストネットワーキングを実装します。

|   #   | 説明                                                                                                                       | レベル | 役割  |
| :---: | ------------------------------------------------------------------------------------------------------------------------ | :-: | :-: |
| 4.3.1 | ネットワークポリシーが、必要なサービスのみを明示的に許可し、それ以外のすべての通信の受信および送信をデフォルトで拒否することを適用していることを確認してください。                                        |  1  | D/V |
| 4.3.2 | 管理者アクセスプロトコル（例：SSH、RDP）およびクラウドメタデータサービスへのアクセスが制限されており、強力な認証を必要とすることを確認してください。                                            |  1  | D/V |
| 4.3.3 | 出口トラフィックが承認された宛先に制限されており、すべてのリクエストがログに記録されていることを確認してください。                                                                |  2  | D/V |
| 4.3.4 | サービス間通信が相互TLSを使用し、証明書の検証および定期的な自動ローテーションを行っていることを確認してください。                                                               |  2  | D/V |
| 4.3.5 | AIのワークロードおよび環境（開発、テスト、本番）が、直接インターネットアクセスや共有IAMロール、セキュリティグループ、環境間の接続を持たない、分離されたネットワークセグメント（VPC/VNet）内で実行されていることを確認してください。 |  2  | D/V |

## C4.4 シークレットおよび暗号鍵管理

秘密情報と暗号鍵を、安全なストレージ、自動ローテーション、および強力なアクセス制御で保護します。

|   #   | 説明                                                                                                             | レベル | 役割  |
| :---: | -------------------------------------------------------------------------------------------------------------- | :-: | :-: |
| 4.4.1 | シークレットが保存時に暗号化され、アプリケーションのワークロードから分離された専用のシークレット管理システムに保存されていることを確認してください。                                     |  1  | D/V |
| 4.4.2 | 暗号鍵がハードウェア対応モジュール（例：HSM、クラウドKMS）で生成および保存されていることを確認してください。                                                      |  1  | D/V |
| 4.4.3 | 本番環境のシークレットへのアクセスには強力な認証が必要であることを確認してください。                                                                     |  1  | D/V |
| 4.4.4 | 秘密情報が専用の秘密管理システムを通じて実行時にアプリケーションに展開されていることを確認してください。秘密情報は決してソースコード、設定ファイル、ビルド成果物、コンテナイメージ、または環境変数に埋め込んではなりません。 |  1  | D/V |
| 4.4.5 | シークレットのローテーションが自動化されていることを確認してください。                                                                            |  2  | D/V |

## C4.5 AIワークロードのサンドボックス化と検証

信頼できないAIモデルを安全なサンドボックス内に分離し、信頼された実行環境（TEE）および機密コンピューティング技術を使用して、機密性の高いAIワークロードを保護します。

|   #   | 説明                                                                                 | レベル | 役割  |
| :---: | ---------------------------------------------------------------------------------- | :-: | :-: |
| 4.5.1 | 外部または信頼されていないAIモデルが隔離されたサンドボックス内で実行されることを確認してください。                                 |  1  | D/V |
| 4.5.2 | サンドボックス化されたワークロードは、デフォルトでアウトバウンドのネットワーク接続がないことを確認し、必要なアクセスは明示的に定義されていることを検証してください。 |  1  | D/V |
| 4.5.3 | モデルまたはワークロードのロード前にワークロードの証明が実施されていることを確認し、信頼された実行環境の暗号的証明を確保してください。                |  2  | D/V |
| 4.5.4 | 機密ワークロードが、ハードウェアにより強制される分離、メモリ暗号化、および整合性保護を提供する信頼実行環境（TEE）内で実行されることを検証してください。      |  3  | D/V |
| 4.5.5 | 秘密の推論サービスが、暗号化された計算を通じて封印されたモデルの重みと保護された実行により、モデルの抽出を防ぐことを確認してください。                |  3  | D/V |
| 4.5.6 | 信頼できる実行環境のオーケストレーションには、ライフサイクル管理、リモート証明、および暗号化通信チャネルが含まれていることを検証してください。            |  3  | D/V |
| 4.5.7 | 安全なマルチパーティ計算（SMPC）が、個々のデータセットやモデルパラメータを公開することなく、協調的なAIトレーニングを可能にすることを検証してください。     |  3  | D/V |

## C4.6 AIインフラストラクチャのリソース管理、バックアップおよびリカバリー

リソース枯渇攻撃を防止し、クォータと監視を通じて公平なリソース配分を確保します。安全なバックアップ、検証済みの復旧手順、および災害復旧機能を通じてインフラの耐障害性を維持します。

|   #   | 説明                                                                                                                                 | レベル | 役割  |
| :---: | ---------------------------------------------------------------------------------------------------------------------------------- | :-: | :-: |
| 4.6.1 | ワークロードのリソース消費が、KubernetesのResourceQuotasなどを使用して適切に制限されていることを検証し、サービス拒否攻撃を緩和する。                                                     |  2  | D/V |
| 4.6.2 | リソース枯渇が、定義されたCPU、メモリ、またはリクエストの閾値を超えた場合に、自動保護機能（例：レート制限やワークロードの分離）が作動することを確認してください。                                                 |  2  | D/V |
| 4.6.3 | バックアップシステムが分離されたネットワーク内で別々の認証情報を使用して動作していること、およびストレージシステムがエアギャップネットワークで運用されているか、無断改ざんを防ぐためにWORM（書き込み一回多数読み取り）保護を実装していることを確認してください。 |  2  | D/V |

## C4.7 AIハードウェアセキュリティ

GPU、TPU、および専用のAIアクセラレータを含むAI専用のハードウェアコンポーネントを保護してください。

|   #   | 説明                                                                                                                       | レベル | 役割  |
| :---: | ------------------------------------------------------------------------------------------------------------------------ | :-: | :-: |
| 4.7.1 | ワークロードの実行前に、ハードウェアベースの認証メカニズム（例：TPM、DRTM、または同等のもの）を使用してAIアクセラレータの完全性が検証されていることを確認してください。                                 |  2  | D/V |
| 4.7.2 | アクセラレータ（GPU）メモリが、ジョブ間のメモリサニタイズを伴うパーティショニング機構を通じてワークロード間で分離されていることを検証してください。                                              |  2  | D/V |
| 4.7.3 | ハードウェアセキュリティモジュール（HSM）が、FIPS 140-3 レベル 3 または Common Criteria EAL4+ の認証を受けて、AIモデルの重みおよび暗号鍵を保護していることを確認してください。             |  3  | D/V |
| 4.7.4 | アクセラレーターファームウェア（GPU/TPU/NPU）がバージョン固定され、署名され、ブート時に証明されていることを検証する。署名されていないファームウェアやデバッグファームウェアはブロックされる。                     |  2  | D/V |
| 4.7.5 | ジョブやテナント間でVRAMおよびオンパッケージメモリがゼロクリアされていること、またデバイスのリセットポリシーがテナント間のデータ残留を防止していることを検証してください。                                  |  2  | D/V |
| 4.7.6 | パーティショニング／アイソレーション機能（例：MIG／VMパーティショニング）が各テナントごとに適用され、パーティション間でのピアツーピアメモリアクセスを防止していることを確認してください。                          |  2  | D/V |
| 4.7.7 | アクセラレータ間接続（NVLink/PCIe/InfiniBand/RDMA/NCCL）が承認されたトポロジーおよび認証されたエンドポイントに限定されていることを確認してください。プレーンテキストのクロステナントリンクは許可されていません。 |  3  | D/V |
| 4.7.8 | アクセラレータのテレメトリ（電力、温度、ECC、パフォーマンスカウンター）がSIEM/OTelにエクスポートされ、サイドチャネルや隠れチャネルを示す異常に関するアラートが発生することを検証してください。                    |  3  |  D  |

## C4.8 エッジおよび分散AIセキュリティ

エッジコンピューティング、フェデレーテッドラーニング、マルチサイトアーキテクチャを含む安全な分散AI展開。

|   #   | 説明                                                                                 | レベル | 役割  |
| :---: | ---------------------------------------------------------------------------------- | :-: | :-: |
| 4.8.1 | エッジAIデバイスが相互TLSを使用して中央インフラストラクチャに認証することを検証してください。                                  |  2  | D/V |
| 4.8.2 | エッジデバイスが検証済み署名付きのセキュアブートおよびロールバック防止機能を実装し、ファームウェアのダウングレード攻撃を防止していることを確認してください。     |  2  | D/V |
| 4.8.3 | 分散型AIの調整が、参加者の検証および悪意のあるノードの検出を伴うビザンチン障害耐性コンセンサスメカニズムを使用していることを確認してください。           |  3  | D/V |
| 4.8.4 | エッジからクラウドへの通信が帯域幅制限、データ圧縮、および暗号化されたローカルストレージを使用したセキュアなオフライン動作をサポートしていることを確認してください。 |  3  | D/V |
