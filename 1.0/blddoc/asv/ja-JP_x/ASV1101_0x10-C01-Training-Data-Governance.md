# C1 トレーニングデータガバナンスとバイアスマネジメント

## 制御目標

トレーニングデータは、出所、セキュリティ、品質、公平性を維持する方法で調達、取り扱い、管理されなければなりません。これにより、法的義務を果たし、トレーニング中に発生する偏り、ポイズニング、またはプライバシー侵害のリスクを低減し、AIのライフサイクル全体に影響を及ぼす可能性を防ぎます。

---

## C1.1 トレーニングデータの出所

すべてのデータセットの検証可能な在庫を維持し、信頼できるソースのみを受け入れ、すべての変更を監査可能な形で記録してください。

|   #   | 説明                                                                                      | レベル | 役割  |
| :---: | --------------------------------------------------------------------------------------- | :-: | :-: |
| 1.1.1 | すべてのトレーニングデータソース（出典、管理者／所有者、ライセンス、収集方法、想定される使用制約、および処理履歴）の最新のインベントリが管理されていることを確認してください。 |  1  | D/V |
| 1.1.2 | トレーニングデータの処理において、不必要な特徴、属性、フィールド（例：未使用のメタデータ、機密の個人識別情報、漏洩したテストデータ）が除外されていることを確認してください。  |  1  | D/V |
| 1.1.3 | すべてのデータセットの変更がログに記録された承認ワークフローの対象であることを確認してください。                                        |  2  | D/V |
| 1.1.4 | 可能な場合は、データセットまたはサブセットにウォーターマークやフィンガープリントが付けられていることを確認してください。                            |  3  | D/V |

---

## C1.2 トレーニングデータのセキュリティと完全性

トレーニングデータへのアクセスを制限し、保存時および転送時に暗号化し、改ざん、盗難、またはデータポイズニングを防ぐためにその整合性を検証します。

|   #   | 説明                                                                                      | レベル | 役割  |
| :---: | --------------------------------------------------------------------------------------- | :-: | :-: |
| 1.2.1 | トレーニングデータの保存およびパイプラインがアクセス制御によって保護されていることを検証してください。                                     |  1  | D/V |
| 1.2.2 | トレーニングデータへのすべてのアクセスが、ユーザー、時間、およびアクションを含めてログに記録されていることを確認してください。                         |  2  | D/V |
| 1.2.3 | トレーニングデータセットが、業界標準の暗号アルゴリズムおよびキー管理の手法を用いて、転送中および保存時に暗号化されていることを検証してください。                |  2  | D/V |
| 1.2.4 | トレーニングデータの保存および転送時に、データの整合性を確保するために暗号学的ハッシュまたはデジタル署名が使用されていることを確認してください。                |  2  | D/V |
| 1.2.5 | トレーニングデータの無断修正や破損を防ぐために、自動検出技術が適用されていることを確認してください。                                      |  2  | D/V |
| 1.2.6 | 古くなったトレーニングデータが安全に消去または匿名化されていることを確認してください。                                             |  2  | D/V |
| 1.2.7 | すべてのトレーニングデータセットのバージョンが一意に識別され、不変の状態で保存され、ロールバックやフォレンジック分析をサポートするために監査可能であることを検証してください。 |  3  | D/V |

---

## C1.3 トレーニングデータのラベリング品質、整合性、およびセキュリティ

ラベルを保護し、重要なデータに対して技術的なレビューを必須とします。

|   #   | 説明                                                                                             | レベル | 役割  |
| :---: | ---------------------------------------------------------------------------------------------- | :-: | :-: |
| 1.3.1 | ラベルアーティファクトの整合性と真正性を確保するために、暗号学的ハッシュまたはデジタル署名が適用されていることを検証してください。                              |  2  | D/V |
| 1.3.2 | ラベリングインターフェースおよびプラットフォームが強力なアクセス制御を実施し、すべてのラベリング活動の改ざん検知可能な監査ログを維持し、無許可の変更から保護していることを確認してください。 |  2  | D/V |
| 1.3.3 | ラベル内の機密情報が、保管時および転送時にデータフィールドレベルで削除、匿名化、または暗号化されていることを確認してください。                                |  3  | D/V |

---

## C1.4 トレーニングデータの品質とセキュリティ保証

自動検証、手動のスポットチェック、および記録された修復作業を組み合わせて、データセットの信頼性を保証します。

|   #   | 説明                                                                                                                                                                                | レベル | 役割  |
| :---: | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :-: | :-: |
| 1.4.1 | 自動テストが、すべての取り込みまたは重要なデータ変換時にフォーマットエラーおよびヌル値を検出することを確認してください。                                                                                                                      |  1  |  D  |
| 1.4.2 | LLMのトレーニングおよびファインチューニングパイプラインが、ポイズニング検出およびデータ整合性検証（例：統計的手法、外れ値検出、埋め込み分析）を実装して、潜在的なポイズニング攻撃（例：ラベルフリッピング、バックドアトリガー挿入、役割切り替えコマンド、影響力のあるインスタンス攻撃）や学習データの意図しないデータ破損を識別していることを確認してください。 |  2  | D/V |
| 1.4.3 | 自動生成されたラベル（例：LLMや弱教師あり学習によるもの）が、幻覚的、不正確、または低信頼のラベルを検出するための信頼度閾値や一貫性チェックの対象となっていることを検証してください。                                                                                      |  2  | D/V |
| 1.4.4 | 生成された敵対的サンプルを使用した敵対的トレーニング、摂動入力によるデータ増強、またはロバスト最適化技術などの適切な防御策が、リスク評価に基づいて関連するモデルに対して実装され、調整されていることを確認してください。                                                                      |  3  | D/V |
| 1.4.5 | すべてのデータ取り込みまたは重要なデータ変換時に、自動テストがラベルのずれを検出することを確認してください。                                                                                                                            |  3  |  D  |

---

## C1.5 データリネージュおよびトレーサビリティ

監査可能性とインシデント対応のために、各データポイントのソースからモデル入力までの完全な経路を追跡してください。

|   #   | 説明                                                                                                     | レベル | 役割  |
| :---: | ------------------------------------------------------------------------------------------------------ | :-: | :-: |
| 1.5.1 | 各データポイントの系統、すべての変換、増強、統合を含めて記録されており、再構築可能であることを確認してください。                                               |  2  | D/V |
| 1.5.2 | 系統の記録が不変で安全に保存され、監査のためにアクセス可能であることを検証してください。                                                           |  2  | D/V |
| 1.5.3 | 系統は、プライバシー保護または生成技術を用いて生成された合成データに対しても系譜追跡が適用され、すべての合成データがパイプライン全体で明確にラベル付けされ、実データと区別されていることを確認してください。 |  2  | D/V |
